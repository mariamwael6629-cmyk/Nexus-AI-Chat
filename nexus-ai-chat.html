Transformer Attention — Technical Breakdown

Since I know you prefer mathematical depth over hand-wavy explanations, let me get into the actual mechanics.

The core operation is Scaled Dot-Product Attention:

Attention(Q, K, V) = softmax(QKᵀ / √d_k) · V
Where Q (queries), K (keys), and V (values) are linear projections of the input. The division by √d_k prevents the dot products from growing too large in high-dimensional spaces, keeping gradients stable during training.

Multi-head attention runs this in parallel across h separate heads, each learning different relational patterns — some heads attend to syntactic structure, others to semantic similarity.

How does this compare to the sparse attention approaches like Longformer? I'm building something that needs to handle 100k token contexts.
10:48 AM
N
Nexus AI
Full attention is O(n²) in both time and memory — catastrophic at 100k tokens. Longformer solves this with a combination of:

1. Sliding window attention — each token attends to a fixed window of neighbors (e.g., ±512). O(n·w) complexity.
2. Global attention tokens — designated tokens (like [CLS]) attend to the full sequence, preserving task-critical global context.

For your use case building at 100k context, I'd also look at FlashAttention-3 which achieves full-attention quality at reduced memory via IO-aware tiling. Since you're on TypeScript/React, the @xenova/transformers WASM port supports this natively.
